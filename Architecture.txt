================================================================
SHARED ACROSS ALL MODELS
================================================================

Dataset        : CHB-MIT Scalp EEG Database (PhysioNet)
                 23 pediatric epilepsy patients
                 ~1,000 hours of EEG · 198 labeled seizure events
Task           : Binary classification — seizure (1) or normal (0) per window
Split          : Patient-level — train on chb01–16, val on chb17–19, test on chb20–24
                 The model is evaluated on patients it has NEVER seen during training.
                 This is the scientifically correct approach — window-level splits
                 leak data because windows from the same patient appear in both
                 train and test, letting the model memorize patient-specific patterns.

Preprocessing:
  - 5-second windows at 256 Hz → 1280 samples per window per channel
  - 50% overlapping stride (2.5s) → more training samples, helps class imbalance
  - A window is labeled seizure if ≥50% of its duration overlaps an annotation
  - Z-score normalization per channel per window (mean=0, std=1)
    Handles the large amplitude variation across electrodes and patients

Input shape    : (batch, 23, 1280) — 23 channels × 1280 time steps

Class imbalance:
  Seizure windows are ~0.5–5% of all windows depending on the patient.
  Without intervention a model learns to always predict "normal" and gets
  98%+ accuracy but 0% recall — clinically useless.

  Handled by:
    1. pos_weight in BCEWithLogitsLoss = neg_count / pos_count (~20–200)
       Penalizes missing a seizure proportionally to its rarity
    2. 50% overlapping windows → ~2× more seizure windows per event
    3. Data augmentation (noise + channel dropout) on training set only

Loss function  : BCEWithLogitsLoss with pos_weight
  Sigmoid + binary cross entropy fused into one numerically stable operation.
  Applies pos_weight to multiply the loss for positive (seizure) samples.

Optimizer      : AdamW, lr=0.001, weight_decay=0.0001
  AdamW decouples weight decay from gradient scaling — standard modern choice.

LR Scheduler   : CosineAnnealingLR, T_max=epochs, eta_min=1e-6
  Smoothly decays LR from initial to near-zero over training.
  Prevents loss from bouncing near convergence.

Gradient clipping : max_norm=1.0
  Critical for LSTM/GRU — prevents exploding gradients during BPTT.
  If loss suddenly spikes or goes NaN, this is likely the cause.
  Set grad_clip: 1.0 in config — do not disable for recurrent models.

Early stopping : patience=10 epochs on val F1 (configurable to val recall)
Primary metric : Recall (sensitivity) — missing a seizure is clinically dangerous.
                 F1-score as balanced secondary metric.

Output         : Raw logit (batch, 1). Apply sigmoid to get probability.
                 BCEWithLogitsLoss handles sigmoid internally during training.
                 threshold in config converts probability → binary prediction.


================================================================
MODEL 1 — CNN BASELINE (Ablation)
================================================================

Type    : No temporal modeling
Purpose : Performance floor. Everything else should beat this.
          If temporal models don't outperform it, the window is too short
          or local EEG morphology alone is sufficient for detection.

Architecture:
  Input         : (batch, 23, 1280)
  Conv Block 1  : Conv1d(23→32, k=5) → BatchNorm1d → ReLU → MaxPool1d(2) → Dropout
  Conv Block 2  : Conv1d(32→64, k=5) → BatchNorm1d → ReLU → MaxPool1d(2) → Dropout
  Conv Block 3  : Conv1d(64→128, k=5) → BatchNorm1d → ReLU → MaxPool1d(2) → Dropout
  Pool          : AdaptiveAvgPool1d(1)  →  (batch, 128, 1)
  Classifier    : Flatten → Linear(128→64) → ReLU → Dropout → Linear(64→1)
  Output        : (batch, 1)  raw logit

Time dimension after each MaxPool: 1280 → 640 → 320 → 160 → 1 (via AdaptiveAvgPool)
After AdaptiveAvgPool: ALL temporal ordering information is discarded.
The model answers "were there seizure-like shapes anywhere?" not
"were those shapes building progressively over the 5 seconds?"

Conv1d kernel_size=5: each filter looks at 5/256 ≈ 20ms of signal at once.
  Larger kernel → wider local context per filter.
  Must be odd (for symmetric padding to preserve output length).

BatchNorm1d after each Conv:
  Stabilizes training by normalizing activations within each batch.
  Critical for EEG because raw amplitude varies hugely across channels.
  Remove it and you'll likely see unstable loss curves.

What to change (configs/config.yaml → cnn_baseline):
  out_channels: [32,64,128]    add entries or increase values for capacity
  kernel_size: 5               larger = wider local temporal context per filter
  dropout: 0.3                 increase to 0.5 if train metrics >> val metrics

MODEL 2 — CNN + LSTM (Primary Model)
================================================================

Type    : CNN for local features + LSTM for temporal evolution
Purpose : Test the core hypothesis: temporal sequential modeling improves seizure detection.
          CNN and LSTM are designed to complement each other.

Architecture:
  Input         : (batch, 23, 1280)
  CNN block     : Identical 3-block CNN as baseline — (batch, 128, 160)
  Permute       : (batch, 160, 128)  — reformat for LSTM: 160 time steps × 128 features
  LSTM          : input=128, hidden=128, layers=2, batch_first=True
  Extract h_n   : final hidden state (batch, 128)
  Classifier    : Linear(128→64) → ReLU → Dropout → Linear(64→1)
  Output        : (batch, 1)  raw logit

Why CNN + LSTM:
  The CNN is the "feature detector" — learns to recognize local EEG patterns
  (spike shapes, sharp waves, rhythmic discharges) at each compressed time step.

  The LSTM is the "pattern tracker" — reads those 160 feature frames in order,
  maintaining a hidden state that summarizes what it has seen so far.
  This is what lets the model detect SEIZURE ONSET: the transition from
  normal EEG patterns to the evolving rhythmic discharges of an ictal event.

LSTM gates (the core mechanism):
  Forget gate : how much of the previous hidden state to discard
  Input gate  : how much new information to add to memory
  Output gate : what to output from current memory state
  These gates let the LSTM selectively retain patterns over 160+ time steps.

Why LSTM over plain RNN:
  Plain RNN gradients vanish after ~10 steps — can't learn long dependencies.
  LSTM's gating prevents this: the forget gate can be set near 1 to preserve
  gradients, enabling stable training over 160 time steps.

Bidirectional (configs/config.yaml → cnn_lstm.bidirectional):
  false (default): model only sees past context — suitable for real-time use
  true: model sees forward and backward — better offline accuracy, doubles params

What to change:
  lstm_hidden: 128    → 256 for more capacity, 64 for faster training
  lstm_layers: 2      → 1 is often enough for 5s windows
  bidirectional: true → for offline analysis / academic comparison


================================================================
MODEL 3 — CNN + GRU (Faster Variant)
================================================================

Type    : CNN + GRU — structurally identical to CNN+LSTM, GRU replaces LSTM
Purpose : Ablation — does LSTM's extra complexity over GRU actually help?

GRU vs LSTM:
  GRU has 2 gates (reset, update) vs LSTM's 3 (input, forget, output).
  GRU has NO separate cell state — just the hidden state.
  Result: ~30% fewer parameters than LSTM, similar performance on most tasks.
  Trains faster. If CNN+LSTM ≈ CNN+GRU in your results, use GRU for deployment.

Key code difference from CNNLSTM:
  LSTM forward: output, (h_n, c_n) = lstm(x)   → need both h_n and c_n
  GRU  forward: output, h_n        = gru(x)     → only h_n (no cell state)

What to change:
  Same knobs as LSTM: gru_hidden, gru_layers, bidirectional, dropout

MODEL 4 — TCN (Temporal Convolutional Network)
================================================================

Type    : Dilated causal convolutions with residual connections
Purpose : Fully parallel alternative to recurrent models.
Based on: Bai et al. (2018) https://arxiv.org/abs/1803.01271

Architecture:
  Input              : (batch, 23, 1280)
  TemporalBlock × 4  : stacked with dilation = 2^i (1, 2, 4, 8)
  AdaptiveAvgPool1d  : collapse time → (batch, 64, 1)
  Classifier         : Flatten → Linear(64→64) → ReLU → Dropout → Linear(64→1)
  Output             : (batch, 1)

Key concept — Dilated Causal Convolution:
  "Causal" = position t only looks at positions ≤ t (no future leakage).
    Implemented by left-padding then chopping the right (_Chomp1d).

  "Dilated" = filter skips every d-1 elements:
    dilation=1: looks at [t-4, t-2, t]       (kernel_size=3, standard)
    dilation=2: looks at [t-8, t-4, t]       (twice as wide, same params)
    dilation=4: looks at [t-16, t-8, t]
    dilation=8: looks at [t-32, t-16, t]

  After 4 blocks: receptive field = (2^4 - 1) × (kernel_size - 1) + 1 = 31 steps

Residual connections (like ResNet):
  Each block: output = ReLU(conv_out + identity_or_1x1_conv(input))
  Prevents vanishing gradients in deep stacks.
  1×1 Conv used as residual branch only when input/output channels differ.

TCN vs LSTM:
  TCN advantage:     all time steps computed simultaneously → faster GPU training
  TCN advantage:     stable gradients (residual connections)
  TCN advantage:     fixed, interpretable receptive field
  LSTM advantage:    adaptive memory — not constrained to a fixed RF size
  LSTM advantage:    may capture very long dependencies better

  For 5s EEG windows at 256Hz with 4 blocks + kernel_size=3:
    RF = 31 compressed time steps. If results are worse than LSTM,
    add more blocks in config.yaml → tcn.num_channels.

What to change (configs/config.yaml → tcn):
  num_channels: [64,64,64,64]    add more entries to grow receptive field
  kernel_size: 3                 5 or 7 for faster RF growth per block
  dropout: 0.2                   TCN is less prone to overfit; keep lower


================================================================
WHY THESE FOUR MODELS
================================================================

The experiment compares along two axes:

  AXIS 1: Does temporal modeling help?
          CNN baseline  vs  CNN+LSTM, CNN+GRU, TCN
          If yes: temporal modeling adds value for seizure onset detection
          If no:  local morphology alone is sufficient at 5s resolution

  AXIS 2: Which temporal approach works best?
          CNN+LSTM  vs  CNN+GRU  →  is LSTM's cell state worth the extra cost?
          CNN+LSTM  vs  TCN      →  recurrence vs parallelism

Expected outcomes:
  CNN+LSTM ≈ CNN+GRU > CNN baseline   temporal modeling helps
  CNN+GRU trains faster with similar accuracy   GRU is preferred for deployment
  TCN competitive with LSTM/GRU               parallelism is viable

If CNN baseline wins:
  → 5s windows may already capture enough morphological context
  → Try shorter windows (1-2s) where temporal structure is more diagnostic

If TCN lags:
  → RF may be too small (31 steps with default config)
  → Add more blocks: num_channels: [64,64,64,64,64,64]

================================================================
METRICS
================================================================

Primary   : Recall (sensitivity) = TP / (TP + FN)
  Missing a seizure (FN) can result in injury, prolonged ictal state, or SUDEP.
  Clinical minimum: recall ≥ 0.90

Secondary : F1-score = 2 × (precision × recall) / (precision + recall)
  Balanced metric. Used for early stopping and model selection.

Supporting:
  Precision     = TP / (TP + FP)  — false alarm rate
  Specificity   = TN / (TN + FP)  — how often we correctly identify normal EEG
  ROC-AUC       — discrimination ability, threshold-independent
  PR-AUC        — more reliable than ROC-AUC under class imbalance (< 5% positive)

Threshold:
  Default 0.5. Lower → higher recall, lower precision.
  Use the threshold_sweep plot to pick your operating point based on
  your clinical recall target.
